大致思路大致就是，一遍扫描前缀和，边扫描边记录，哪些地方可以切第一刀，哪些地方可以切第二刀。如果位置j可以切第二刀，那么它与前面第一刀进行组合，就可以切成三部分。

切第一刀的判断条件是：s[i]==t；

​ 处理方法是：cnt++（cnt的意思就是i位置前面有多少可以切第一刀的地方）

切第二刀的判断条件是：s[n]-s[i+1]==t

​ 处理方法是：ans+=cnt，第二刀跟前面的cnt个第一刀组合，所以总个数增加cnt个

Linux入门书籍推荐

入门书籍其实就是对 Linux 系统的基本操作有个了解，这也是很多人对 Linux 学习的基本需求。一般来讲，入门书籍都会讲到以下几方面的内容：

  • Linux 历史
  • 文件及目录管理
  • 文本处理
  • 磁盘管理
  • 进程管理
  • 性能监控
  • 网络工具
  • 用户管理
  • 系统管理
  • Shell编程
  • ……

基于此，推荐一些入门书籍。

git学习

  • git commit

  • git branch

  • git checkout 分支名

  • git merge 在 Git 中合并两个分支时会产生一个特殊的提交记录

  • git rebase 另一种合并分支的方法，可以创造更线性的提交历史

  • 提交树上移动

    学习文章：图解git，用手绘图带你理解git中分支的原理和应用

    • HEAD 是一个对当前检出记录的符号引用 —— 也就是指向你正在其基础上进行工作的提交记录。HEAD 总是指向当前分支上最近一次提交记录
    • 指定提交记录哈希值 git log 来查查看提交记录的哈希值
    • 相对引用：
      • 使用 ^ 向上移动 1 个提交记录
      • 使用 ~<num> 向上移动多个提交记录，如 ~3 （可选，不跟数字时与 ^ 相同，向上移动一次）
    • 相对引用最多的就是移动分支，可以直接使用 -f 选项让分支指向另一个提交

  • 撤销变更，由底层部分（暂存区的独立文件或者片段）和上层部分（变更到底是通过哪种方式被撤销的）组成。

    • git reset
    • git revert 在远程上撤销，会产生一个新的提交（内容是撤销过后的）

  • 整理提交记录

    • git cherry-pick <提交号>... （将一些提交复制到当前所在的位置（HEAD）下，在了解那些提交的哈希值时）

    • 交互式的 rebase（当不清楚想要的提交记录的哈希值，从一系列的提交记录中找到想要的记录）

      • 交互式 rebase 指的是使用带参数 --interactive 的 rebase 命令, 简写为 -i

  • 远程仓库

    • git clone 命令在真实的环境下的作用是在本地创建一个远程仓库的拷贝|在远程创建一个你本地仓库的副本

      • 本地仓库多了一个名为 o/main 的远程分支
      • 远程分支反映了远程仓库(在你上次和它通信时)的状态

    • 从远程仓库获取数据

      • git fetch 从远程仓库获取数据时, 远程分支也会更新以反映最新的远程仓库。（git fetch 并不会改变你本地仓库的状态。它不会更新你的 main 分支，也不会修改你磁盘上的文件，进一步操作可以通过git merge o/main,git cherry-pick o/main ,git rebase o/main）先抓取更新再合并到本地分支

      • git pull一步完成先抓取更新再合并到本地分支

    • 向远程仓库传输数据

      • git push 负责将你的变更上传到指定的远程仓库，并在远程仓库上合并你的新提交记录

  • 处理偏离的提交历史

    • git fetch–>git rebase o/main（可能顺便处理冲突）–>git push 【简写（fetch 和 rebase）git pull --rebase】

    • git fetch–>git merge–>git push

  • 新建一个分支feature, 推送到远程服务器. 然后reset你的main分支和远程服务器保持一致
