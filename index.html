这些变量是可选的，用于创建新用户和设置该用户的密码。该用户将被授予MYSQL_DATABASE变量指定的数据库的超级用户权限（见上文）。要创建用户，这两个变量都是必需的。
请注意，无需使用此机制来创建超级用户超级用户，默认情况下会使用MYSQL_ROOT_PASSWORD变量指定的密码创建该用户。

MYSQL_RANDOM_ROOT_PASSWORD

这是一个可选变量。设置为非空值，例如yes，为 root 用户生成一个随机初始密码（使用pwgen）。生成的 root 密码将打印到 stdout ( GENERATED ROOT PASSWORD: …..)。

HTML

若想要理清其中的脉络，不如从技术的发展来看。
首先我们要跟随HTML的脚步出发，HTML是Hypertext Markup Language（超文本标记语言）的缩写，是Web用于编辑网页的主要工具。
浏览器通过读取html文件，生成dom树，将信息展示在页面上。
我们所看到的各个网页，均为一个个html文件，而html文件，说白了也只是一个很长字符串。
所以说，我们查看的网页均为浏览器读取这个html文件内的字符串所产生的
那html文件的内容是如何来的呢？

静态页面

然后把该html文件放置在服务器内，谁想请求查看这篇文章的网页

浏览器接收到此html文件后，解析生成网页，一切就完成了。

Thrift的使用实践

Posted on 2022-01-09 In Thrift

Thrift介绍

RPC(Remote Procedure Call)，远程过程调用，）是一个计算机通信协议，此协议允许进程间通信。

简单来说，当机器 A 上的进程调用机器 B 上的进程时，A 上的调用进程被挂起，而 B 上的被调用进程开始执行。

调用方可以通过参数将信息传送给被调用方，然后可以通过被调用方传回的结果得到返回。

RPC 框架屏蔽了底层传输方式（TCP/UDP）、序列化和反序列化（XML/JSON/二进制）等内容，使用框架只需要知道被调用者的地址和接口就可以了，无须额外地为这些底层内部编程。

大部分的RPC框架都遵循如下三个开发步骤：

  1. 定义一个接口说明文件：描述了对象(结构体)、对象成员、接口方法等一系列信息；
  2. 通过RPC框架所提供的编译器，将接口说明文件编译成具体的语言文件；
  3. 在客户端和服务器端分别引入RPC编译器所生成的文件，即可像调用本地方法一样调用服务端代码；
