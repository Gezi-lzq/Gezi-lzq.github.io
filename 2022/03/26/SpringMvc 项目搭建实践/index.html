关于Add frameworks (facets)

为了开发特定于框架的应用程序，IntelliJ IDEA 具有方面。构面包含库、依赖项和技术，它们为您提供额外的 UI 元素来配置特定于框架的设置。
IntelliJ IDEA 可以识别特定框架的典型文件或目录，并为您您添加必要的方面。一旦检测到并添加了 facet，IntelliJ IDEA 将通知您缺少的配置并建议必要的操作。

手动向模块添加facet

  1. 在项目工具窗口Alt+1中，右键单击要添加facet的模块，然后选择添加框架支持。
  2. 从列表中选择必要的框架。根据您的选择，系统可能会提示您配置其他设置（例如，配置库）。
  3. 应用更改并关闭对话框。

Upon initialization of a DispatcherServlet, Spring MVC looks for a file named [servlet-name]-servlet.xml in the WEB-INF directory of your web application and creates the beans defined there, overriding the definitions of any beans defined with the same name in the global scope.
在初始化 DispatcherServlet 时，Spring MVC 会在您的 Web 应用程序的 目录中DispatcherServlet查找一个名为[servlet-name]-servlet.xml 的文件，并创建在那里定义的 bean，覆盖在全局范围内有相同名称的bean。

配置Tomcat服务器

一定要使用Tomcat9：
原因如下：

Spring（和 Spring Boot）目前仅支持 JavaEE，不支持 JakartaEE。这意味着它只能在与 JavaEE 兼容的服务器上运行。Tomcat10是jakartaee规范的实现，因此目前不被spring支持的
唯一的解决方案是降级到 Tomcat 9。
